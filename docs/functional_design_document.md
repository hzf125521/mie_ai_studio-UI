# MIE AI Studio - 功能设计文档

## 1. 概述
MIE AI Studio 是一款专注于工业设备（如旋转设备、泵站等）运行状态监控与异常检测的前端应用。系统通过引导式的操作流程（Step-by-Step），支持用户进行设备选择、任务类型选择、信号数据准备、模型训练、模型验证以及最终的模型部署与实时监控。

---

## 2. 页面功能与操作说明

### 2.1 设备与模型管理主页 (Device Selection)
**功能描述：**
作为系统的入口页面，主要用于设备选择和全局模型管理。用户可以指定需要监控的设备，并查看该设备下或所有设备的历史模型及部署状态。

**操作说明：**
- **设备选择：** 在下拉框中选择目标设备（如 Pump Station A），点击“配置模型 (Configure Model)”进入模型配置工作流（步骤 1）。
- **模型列表查看：** 展示所有已创建的算法模型，包括设备名称、算法类型、模型名称、当前状态（训练中 / 已训练 / 已验证 / 已部署）等。
- **模型操作：**
  - **详情 (Details)：** 点击后跳转至该模型的训练/验证/部署详情页。
  - **部署 / 停止 (Deploy / Stop)：** 对于已验证 (Validated) 的模型，可点击“部署”将其部署到生产环境；若已部署，则可点击“停止”停止部署。
  - **删除 (Delete)：** 删除不需要的模型。

---

### 2.2 步骤 1: 任务类型选择 (Workflow Selection)
**功能描述：**
进入模型配置工作流的第一步，用户需确定针对所选设备要执行的 AI 检测任务类型。

**操作说明：**
- **异常检测 (Outliers Detection)：** 点击选择该模式。适用于使用无监督学习模型（如 Autoencoder, Isolation Forest）检测时间序列数据中的异常和离群点。
- **回归残差 (Regression Residual)：** 点击选择该模式。适用于基于回归模型（如 RandomForest）监控残差，通过预测误差和阈值来设定报警。

---

### 2.3 步骤 2: 信号准备 (Step1.tsx)
**功能描述：**
用于准备和筛选用于模型训练或验证的传感器信号数据。用户可以选择不同的测点特征，预览数据，并将其保存为信号样本。

**操作说明：**
- **特征选择：** 在左侧边栏，从不同的测点（如 Point A, Point B）中勾选需要分析的特征（如 Temp, Pressure）。若选择了回归任务，还需额外指定一个 Target Feature (y)。
- **时间范围设置：** 设定提取信号的开始与结束时间。
- **信号筛选预览：** 点击“信号筛选预览”按钮，系统会在右侧图表区生成并展示该配置下的模拟数据（带有“预览”标签），供用户确认数据形态（此时数据未正式保存）。
- **添加信号样本：** 确认无误后，点击“添加信号样本”，将当前预览或配置的数据正式保存到系统的信号列表中。
- **信号列表管理：** 左下角展示已保存的信号列表，支持搜索、按日期过滤、重命名（点击编辑图标）以及删除（前提是未被模型使用）。
- **可视化展示：** 点击列表中的信号，右侧会通过时间序列图、2D PCA图或 3D 散点图展示信号的数据分布。

---

### 2.4 步骤 3: 模型训练、验证与部署 (StepModelTrainingValidation.tsx)
**功能描述：**
核心的 AI 操作空间，整合了模型训练（Training）、模型验证（Validation）和模型部署（Deployment）三个阶段。顶部提供 Tab 切换。

#### 2.4.1 模型训练 (Model Training)
**操作说明：**
- **新建模型：** 点击“新建模型”，在弹窗中选择步骤 2 中准备好的信号样本。
- **配置参数：** 设置数据预处理选项（如标准化、PCA 降维），选择具体算法（根据任务类型不同，可选 RandomForest、Autoencoder 等），并配置算法的超参数（如 Epochs, Batch Size 等）。
- **开始训练：** 提交后系统开始模拟训练。训练完成后，右侧面板会展示模型的评估指标（如 R² Score 或 ROC/Precision），并通过多种图表（真实值 vs 预测值、残差分布、特征重要性、异常得分等）可视化训练结果。

#### 2.4.2 模型验证 (Model Validation)
**操作说明：**
- **新建验证：** 在左侧选中一个状态为已训练的模型，点击“新建验证”。
- **选择验证数据：** 在弹窗中选择与训练集特征对齐的新信号样本进行验证。
- **查看验证结果：** 验证完成后，页面会展示验证集的指标和可视化图表，帮助评估模型在新数据上的泛化能力。支持保存多次验证记录并可进行对比查看。

#### 2.4.3 模型部署与监控 (Model Deployment)
**操作说明：**
- **部署配置：** 选择一个已通过验证的模型，可自定义报警文案（当检测到异常时触发的提示信息）。
- **执行部署：** 点击“部署到生产环境”将模型激活。
- **实时监控预览：** 部署成功后，点击“实时监控预览”，页面会进入动态模式，通过折线图、散点图等实时滚动展示系统模拟的设备运行数据、残差变化或异常得分情况，直观呈现 AI 模型的在线监测效果。

---

## 3. 架构设计亮点
- **状态共享与隔离：** 通过 `AppContext` 集中管理设备、信号、模型和验证的全局状态，确保跨步骤的数据一致性。
- **高复用可视化组件：** 抽象了统一的 Chart 组件（如 PCA, 3D 散点, 时序图），在信号预览、模型训练、验证及实时监控等多个场景中高度复用。
- **向导式交互：** 步骤清晰明确，校验严格（如必须先验证模型才能部署），极大降低了工业 AI 模型落地的操作门槛。
